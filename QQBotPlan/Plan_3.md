## 一、优化背景

### 成本现状（3个活跃群 × 16h/天）

| 模型 | 用途 | 日调用 | 日成本 | 月成本 | 占比 |
|------|------|--------|--------|--------|------|
| FlashLite (flash-lite) | 消息判断+压缩 | 368 | $0.022 | ¥4.7 | 10% |
| **主模型 (flash-preview)** | 回复生成 | 50 | **$0.130** | **¥28.1** | **60%** |
| 工具模型 (flash-preview) | 后台任务 | 5 Task | $0.064 | ¥13.7 | 30% |
| **合计** | | | **$0.215/日** | **¥46.5/月** | 100% |

### 关键发现
- **FlashLite 不是成本瓶颈**：虽然调用频率最高，但使用 flash-lite 模型（单价 1/10），实际成本占比仅 10%
- **主模型是绝对大头**：flash-preview 单价高 10×，加上思考 token（$0.40/M）和巨大上下文（10K+ input/次），占 60%
- **本地部署不划算**：仅电费就 ¥67.5/月，远超 API 成本，且运维成本不可忽略
- **KV Cache 已在帮忙**：FlashLite 的隐式缓存命中率约 90%，对其 10% 的占比进一步压缩

### 优化策略
- ❌ ~~本地 Gemma 4 替代 FlashLite~~ — 已排除（成本不划算）
- ✅ FlashLite 智能采样优化 — Plan_3_1
- ✅ FlashLite+主模型 KVCache 静态/动态分离 — Plan_3_2
- ✅ 工具模型 KVCache 静态/动态分离 — Plan_3_3
- ✅ 面板可控性 + 成本监控 — Plan_3_4

## 四、技术基座确认

### Gemini API 关键能力

| 能力 | 支持情况 | 用途 |
|------|---------|------|
| 隐式缓存 | ✅ Gemini 2.5+ 自动启用，按前缀匹配 | system_instruction 静态化 → 自动命中 |
| 显式缓存 | ✅ 付费层级支持 (systemInstruction+contents+tools) | 可选升级方案（当前用隐式即可） |
| `countTokens` API | ✅ `POST models/{model}:countTokens` | 精确 token 计算（替代本地估算） |
| `usageMetadata` 响应 | ✅ 含 `cached_content_token_count` | 每次调用获取缓存命中 token 数 |
| AstrBot 框架支持 | ✅ `gemini_source.py` L530 已提取 `input_cached` | 无需额外开发 |

### 隐式缓存最佳实践（来自官方文档）
- "尝试将较大且常见的内容放置在提示的**开头**"
- "尝试在短时间内发送具有相似前缀的请求"
- 各模型有最小输入 token 阈值（详见 Plan_3_2）

## 五、讨论决策记录

### 2026-04-11 本地部署 vs API 优化讨论

**结论**：放弃本地部署方案，转向 API 层面的效率优化。

**理由**：
1. FlashLite 月成本仅 ¥4.7，本地部署电费 ¥67.5/月 = 高 14 倍
2. 即使消息量增 20× + KV Cache 命中率降至 50%，FlashLite 月成本也仅 ~¥40-50，仍低于本地部署
3. API 方案零运维、高可靠、弹性伸缩，本地部署需要专机 24h 运行
4. 真正的成本大头（60%）在主模型 flash-preview，优化那边更有效

### 2026-04-13 Plan_3 方案定稿讨论

**核心决策**：
1. **方案 A（全放 systemInstruction）** 确认为主模型缓存方案 — 静态 Section 全放 system_instruction，动态部分拼到 user message 前缀
2. **Plan_3_3 独立**：工具模型的 KVCache 分离单独成文件，不合并进 Plan_3_2
3. **成本监控**：使用本地 `usageMetadata` 记账 + 官方 `countTokens` API 获取精确 token 数，不依赖 GCP 控制台
4. **工具模型不降级**：维持 flash-preview，不引入复杂的模型选择/智能终止机制
5. **Sandbox 环境（Section 16）当动态处理**：虽然运行时很少变化，但保守起见不放入缓存区
