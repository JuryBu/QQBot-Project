## 一、当前机制分析

### 触发逻辑（main.py L608-623）

```
群聊普通消息 → 计数器 +1
  if 计数器 >= sync_interval(默认5):  → 触发 FlashLite
  elif 距上次 >= 60s AND 计数器 >= 1:  → 触发 FlashLite（时间兜底）
  
群聊 @/唤醒词 → 立即异步触发（不受间隔限制）
私聊 → 每条消息触发
```

### 问题
1. sync_interval 全局统一（所有群相同），无法按群活跃度差异化
2. 时间兜底条件"至少1条消息"过于宽松，闲时一条无意义消息也会触发
3. 60s 时间间隔和 1 条消息阈值都是硬编码
4. 没有基于活跃度的智能调节机制

## 二、改进方案

### 2.1 面板参数化（基础）

将以下硬编码值全部暴露到面板：

| 参数 | 当前值 | 建议默认值 | 面板字段名 | 说明 |
|------|--------|-----------|-----------|------|
| sync_interval | 5 | 5 | 同步间隔(条) | ✅ 已有 |
| sync_time_interval | 60s | 60s | 时间兜底间隔(秒) | 🆕 需新增 |
| sync_time_min_msgs | 1 | 3 | 时间兜底最低消息数 | 🆕 需新增 |

**代码改动**：
- `self._sync_time_min_msgs = self._cfg("sync_time_min_msgs", 3)`
- L614 改为: `time_trigger = time_elapsed >= self._sync_time_interval and self._msg_counters[group_id] >= self._sync_time_min_msgs`
- 面板 config_schema + 前端对应

### 2.2 每群独立配置

在面板增加"群聊配置覆盖"区域，支持按群号设置独立的 sync_interval：

```json
{
  "group_overrides": {
    "123456789": { "sync_interval": 10 },  // 活跃群：间隔大
    "987654321": { "sync_interval": 3 }    // 不活跃群：间隔小
  }
}
```

**取值优先级**：群独立配置 > 智能动态 > 全局默认

### 2.3 智能动态采样（高级）

根据群聊活跃度自动调整 sync_interval，无需手动配置每个群：

#### 算法设计

```python
def _calc_dynamic_interval(self, group_id: str) -> int:
    """根据群活跃度动态计算采样间隔"""
    # 统计过去 10 分钟的消息量
    recent_msgs = self._recent_msg_rates.get(group_id, 0)  # msgs/10min
    
    # 活跃度分级
    if recent_msgs >= 30:      # 非常活跃（3+ msg/min）
        return 15              # 每15条采样一次
    elif recent_msgs >= 15:    # 活跃（1.5+ msg/min）
        return 10              # 每10条采样一次  
    elif recent_msgs >= 5:     # 普通活跃
        return 5               # 每5条采样一次（默认）
    else:                      # 不活跃（< 0.5 msg/min）
        return 3               # 每3条采样一次（更细）
```

#### 核心思路
- **活跃群间隔大**：信息密度高，采样几条就能概括全貌
- **不活跃群间隔小**：每条消息都可能是有价值的独立话题，不能漏
- 活跃度统计用滑动窗口（10分钟），避免突发消息导致的误判
- 面板增加"动态模式"开关（默认开启），关闭后退回全局固定值

### 2.4 面板开关：动态模式

```
FlashLite 采样策略:
  ○ 固定模式（使用全局同步间隔）  
  ● 动态模式（根据群活跃度自动调整）   ← 推荐
  
动态模式参数:
  非常活跃阈值(条/10min): [30]  → 间隔 [15]
  活跃阈值(条/10min):     [15]  → 间隔 [10]
  普通阈值(条/10min):     [5]   → 间隔 [5]
  不活跃(低于普通阈值):          → 间隔 [3]
```

## 三、成本影响估算

### 场景：3个群（1活跃+1普通+1不活跃），16h/天

| 方案 | 活跃群 | 普通群 | 不活跃群 | 日触发 | 月成本 |
|------|--------|--------|---------|--------|--------|

| 动态模式 | 10条/触发 | 5条/触发 | 3条/触发 | ~230 | ¥2.9 |
