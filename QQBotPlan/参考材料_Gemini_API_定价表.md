## 八、关键设计要点（供成本监控代码参考）

### A. 分级定价模型列表
以下模型有 ≤20万 / >20万 token 的分级定价：
1. `gemini-3.1-pro-preview` → 输入/输出/缓存均分级
2. `gemini-2.5-pro` → 输入/输出/缓存均分级
3. `gemini-2.5-computer-use-preview-10-2025` → 输入/输出分级

### B. 图片输出计价特殊性
以下模型图片输出不走常规 token 计价，按图片张数和分辨率换算 token 数：
1. `gemini-3.1-flash-image-preview` → $60/百万token（图片输出）
2. `gemini-3-pro-image-preview` → $120/百万token（图片输出）
3. `gemini-2.5-flash-image` → $30/百万token（图片输出）

### C. 音频输入/输出特殊计价
Live API 和原生音频模型有独立的音频计价，与文本计价不同。

### D. 面板模型列表是动态获取的
面板下拉列表通过 API 实时获取可用模型，不是硬编码的。模型数量会随 Google API 更新而变化。
定价系统必须通过模糊匹配 + 别名映射来覆盖所有可能出现的模型名。

### E. 缓存存储费注意
我们的系统使用了**显式 Context Caching**（`kv_cache.py` → `createCachedContent` API），
存储费（如 $1.00~$4.50/小时/百万token）是**真实产生**的。当前 `_calc_cost()` 完全没有计入此项。
需要追踪缓存的创建时间、过期时间和 token 规模来估算存储费用。
