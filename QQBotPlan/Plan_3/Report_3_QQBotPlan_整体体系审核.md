## 🔴 严重安全问题

### 安全问题 1: BossLady 控制台无鉴权
- **位置**: `BossLady_Console/backend/main.py:42`, `routers/models.py:71`, `routers/bot.py:145`
- **描述**: 所有 `/api/*` 接口无 session/token 校验，任何人可直接调用高危操作（修改模型、管理 Bot、读取对话）
- **影响**: 🔴 严重 — 控制台暴露在网络中时，任何人都能完全控制
- **修复方案**: 加统一鉴权中间件，所有 `/api/*` 强制校验会话/token
- **备注**: `/system/password` 仅写 hash 不校验，形同虚设

### 安全问题 2: Sandbox 边界失效
- **位置**: `sandbox.py:673/756`, `web_engine.py:268`
- **描述**: `sandbox_exec` 可回退系统解释器执行任意代码，`web_fetch(file://)` 可读本地任意文件
- **影响**: 🔴 严重 — AI 代码执行无隔离，恶意 prompt 可突破 sandbox
- **修复方案**: 
  - 禁止系统解释器回退
  - 限制 `file://` 协议仅允许 sandbox 工作区路径
  - 加进程级隔离

### 安全问题 3: Memory 工作区隔离绕过
- **位置**: `main.py:920/3627`, `memory.py:672/756`
- **描述**: 存在跨窗口读写/召回，workspace 隔离可被绕过
- **影响**: 🔴 中高 — 群 A 可能读到群 B 的记忆
- **修复方案**: 工具层强制注入当前 workspace，存储层无 workspace 直接拒绝

### 安全问题 4: 表情包接口路径穿越
- **位置**: `routers/models.py:616/636/651/673`
- **描述**: 文件路径参数未做边界校验，Windows 下可构造路径穿越
- **影响**: 🔴 中 — 越权读写删本地文件
- **修复方案**: `resolve() + relative_to()` 硬校验 + 文件名白名单 + 上传文件名服务端重写

## 🟡 已知功能缺陷

### 1. `group_overrides` 全链路断裂
- **描述**: 前端可编辑群独立配置，但后端不保存不返回，实质是"假成功"
- **详见**: Report_3_Plan3_对照审核.md

### 2. 前端 onclick 注入风险
- **位置**: `app.js:2011/2021`
- **描述**: `gid` 直接拼入 `onclick` 属性，可构造恶意群号执行 JS
- **修复方案**: 改用 `addEventListener` + 数字白名单
